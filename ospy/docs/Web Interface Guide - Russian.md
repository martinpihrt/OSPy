# Установка OSPy
Рекомендуется чистая установка с актуальной версией Python 3. При первом запуске OSPy создаёт данные для входа. После первого входа пароль и при необходимости имя пользователя нужно изменить в настройках.

## Установочный скрипт
Войдите на Raspberry Pi по SSH и выполните:

```bash
wget https://raw.githubusercontent.com/martinpihrt/OSPy/master/ospy_setup.sh
sudo bash ospy_setup.sh
```

После установки Raspberry Pi перезагрузится, и OSPy будет готов к настройке через браузер.

# Главная страница
Главная страница является основным центром управления. Она показывает часы, навигацию, общие переключатели, временную шкалу, графики, системную информацию и элементы, добавленные расширениями.

## Нормальный уровень воды
Задаёт общий процент уровня воды, который изменяет время работы программ полива.

## Активная задержка дождя
Приостанавливает полив для станций, которые не игнорируют дождь.

## Расписание - ручной режим
Переключает систему между автоматическим расписанием и ручным управлением станциями.

## Включено - выключено
Управляет общей работой OSPy.

## Остановить все станции
Немедленно останавливает выполняющуюся программу или станцию.

## График водного баланса
Показывает количество воды, поданное по станциям или программам.

## Задержка дождя / обнаружен дождь
Когда активен датчик дождя или задержка дождя, соответствующие станции блокируются.

## Системная информация
В нижней части страницы отображаются температура CPU, загрузка CPU, версия ПО, внешний IP-адрес и время работы.

# Программы
Страница программ используется для создания, организации и управления расписаниями полива.

## Добавить новую программу
Создаёт новую программу расписания.

## Группы программ
Программы можно организовать в сворачиваемые группы, например для летних и зимних программ.

## Добавить группу
Создаёт новую группу программ.

## Переименовать группу
Меняет только имя группы. Программы внутри группы остаются без изменений.

## Включить или выключить группу
Включает или выключает все программы в группе одновременно.

## Копировать группу
Создаёт копию группы и её программ. Скопированные программы по умолчанию выключены.

## Удалить группу
Удаление требует подтверждения. Программы из удалённой группы перемещаются в группу по умолчанию.

## Запустить сейчас
Запускает программу немедленно, независимо от расписания.

## Редактировать
Открывает параметры существующей программы.

## Копировать
Создаёт выключенную копию программы для безопасного редактирования.

## Переместить в группу
Назначает программу выбранной группе.

## Удалить всё
После подтверждения удаляет все программы.

## Включить или выключить программу
Переключатель ON/OFF включает или выключает отдельную программу.

## Предупреждения о конфликтах
При сохранении OSPy проверяет включённые программы на пересечение времени для одной и той же станции или выхода. Предупреждение ничего не блокирует автоматически, но показывает пересечение.

## Тип расписания
Доступны выбранные дни, повторение, недельный режим, пользовательский режим и программы на основе прогноза погоды.

## Без корректировок
Для программы не применяются автоматические изменения времени работы.

## Ограничение
Если скорректированное время работы меньше заданного предела, программа будет пропущена.

## Активировать мастер
Станции с настройкой активации мастера по программе используют мастер-станцию, выбранную в программе.

# Расширения
Страница расширений управляет установленными и доступными плагинами.

## Управление
Открывает менеджер расширений, где плагины можно включать, выключать, устанавливать и обновлять.

## Установить новое расширение
Показывает доступные расширения из репозитория.

## Собственный плагин ZIP
Устанавливает плагин из ZIP-файла. Файл должен содержать полную структуру плагина.

Установка или обновление плагина запускает код из выбранного источника. Перед установкой или обновлением OSPy показывает URL репозитория и просит подтверждение. Действие загрузки изменений показывает сообщение проверки, пока OSPy обращается к удаленному репозиторию.

## Выключить всё / Включить всё
Управляет всеми установленными расширениями одновременно.

## Проверка обновлений
Автоматически проверяет наличие новых версий плагинов.

## Загрузить изменения / Изменения
Загружает и показывает последние изменения из репозитория плагинов.

## Автоматические обновления
Если включено, плагины обновляются автоматически. Сначала нужно обновлять OSPy, затем плагины.

# Настройки
Настройки разделены на секции: система, погода, пользователи, безопасность, датчики, обработка станций, мастер-станции, датчик дождя, журналирование, перезапуск, резервная копия и SSL.

## Система
Имя системы, тема, формат времени, HTTP/S адрес и порт, язык, изображения станций и отображение плагинов или датчиков на главной странице.

## Погода
Включает погодные функции, API-ключ и местоположение.

## Пользователи
Управляет входом, паролем и дополнительными пользователями.

## Безопасность
Содержит защиту форм, HTTPS, доменное имя, собственные сертификаты и параметры доступа к API.

### API CORS allowed origin
Этот параметр управляет заголовком `Access-Control-Allow-Origin`, который API использует для клиентов, работающих в браузере. `*` разрешает любой origin, одно значение вроде `https://example.com` разрешает только этот origin, несколько origin можно разделить запятыми, а пустое значение отключает отправку CORS-заголовков. Это не заменяет аутентификацию API; параметр только определяет, какие browser origin могут читать ответы API.

### Enable API JSONP
Этот параметр включает устаревший параметр `callback` для JSONP-ответов API. Оставьте его выключенным, если старая интеграция не требует JSONP. Обычные API-клиенты должны использовать JSON с CORS.

### Remembered browser logins
Страница входа может запомнить браузер с помощью долгосрочного случайного токена, сохраненного в защищенной cookie. OSPy хранит только hash этого токена, а не пароль пользователя. Кнопка Revoke удаляет все запомненные входы; затронутые браузеры должны снова войти с паролем.

## Датчики
Задаёт пароль для загрузки прошивки в датчики.

## Обработка станций
Задаёт максимальное использование, число выходов, паузу между станциями и минимальное время работы. Максимальное использование определяет, могут ли станции работать одновременно. `0` означает без ограничения, `1` означает одну станцию за раз, если у каждой станции использование `1`.

Пауза между станциями вставляется между последовательными запусками, когда планировщик не может запустить станции одновременно. Она не смещает станцию относительно мастер-станции. Пример: при максимальном использовании `1` и использовании станции `1` значение `30` запустит следующую станцию через 30 секунд после окончания предыдущей.

Минимальное время работы может пропустить эту паузу, если предыдущий запуск был короче. Пример: при паузе `30` и минимальном времени работы `10` станция, работавшая только 5 секунд, не вызовет паузу 30 секунд.

## Мастер-станции
Настраивает мастер-станцию 1, мастер-станцию 2 и временные смещения включения и выключения. Мастер-станция обычно является насосом или главным водяным клапаном и используется только для станций, которым она назначена.

Смещение запуска отсчитывается относительно запуска станции. Отрицательные значения запускают мастер раньше, положительные позже. Пример: `-10` запускает мастер за 10 секунд до станции, `+10` запускает его через 10 секунд после станции.

Смещение выключения отсчитывается относительно окончания станции. Отрицательные значения выключают мастер раньше, положительные оставляют его работать дольше. Пример: `-5` выключает мастер за 5 секунд до окончания станции, `+20` выключает его через 20 секунд после окончания станции.

Та же логика применяется к мастер-станции 2, у которой есть собственные настройки смещения запуска и выключения.
